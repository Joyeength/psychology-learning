## Key Files

| File | Role |
|------|------|
| `data.js` | Single source of truth: `MODULES` (id, name, range), `TOPICS` (day → title), `getMod(day)`. Imported by the component. |
| `psych_challenge_2.jsx` | Root component — merges `data.js` MODULES with `MODULE_EXTRAS`, holds all state, renders the 10×10 grid and delegates to `DetailPanel`. |
| `DetailPanel.jsx` | Stateless panel — renders day header, loading/error/content states, mark-done button. Imports `TOPICS` and `Block`. |
| `Block.jsx` | Presentational card — icon + label + text with accent-colored left border. |
| `lessons/[day].json` | One file per day (1–100) with 5 Vietnamese keys. Generated by the content agent pipeline. |
| `.claude/agents/` | 11 agent definitions covering the full product team (PM, Tech Lead, UX, QC, Security, DevOps, Analytics, Technical Writer, and content pipeline). See [project.md](project.md). |

## Critical Constraint

- `MODULE_EXTRAS` in `psych_challenge_2.jsx` is keyed by module **id**: `MODULE_EXTRAS[m.id]`
- Adding a new module to `data.js` requires a matching entry in `MODULE_EXTRAS` with the same numeric id

## Host Environment Requirements

The component expects these to be provided by the host:

- `window.storage` — custom async key/value store
  - `get(key)` → `{ value }`
  - `set(key, value)`
  - **Not** `localStorage`
- `Lucide React` — available as an import
- `React` — available as an import
- Static file serving for `./lessons/` and `./data.js`
